## Overview
The Secure Financial Transaction System is a cybersecurity-focused project designed to simulate, analyze, and secure financial data transmissions within a controlled network environment. The goal is to understand how sensitive financial data flows through systems and how it can be monitored and protected from potential cyber threats such as packet sniffing, unauthorized access, and data leakage.

This project demonstrates practical skills in network analysis, Linux security tools, and traffic inspection using PCAP files.

**Objectives**
1. Simulate a secure financial transaction environment
2. Capture and analyze network traffic using PCAP files
3. Identify potential vulnerabilities in data transmission
4. Strengthen understanding of packet-level communication
5. Apply cybersecurity best practices in a Linux environment
   
**Tools & Technologies Used**
- Linux (Kali / Ubuntu)
- Bash Scripting
- Git & GitHub
- Wireshark (PCAP Analysis)
- tcpdump (Packet Capture)
- Python
  
**Project Workflow**
1. Environment Setup
- Configured Linux-based system for analysis
- Installed required networking tools
2. Traffic Generation
- Simulated financial transaction data flow
- Generated network traffic for monitoring
3. Packet Capture
- Captured network packets using tcpdump / Wireshark
- Stored data in .pcap format
4. Traffic Analysis
- Analyzed packets using Wireshark
- Inspected protocols, sources, and data flow behavior
5. Security Observation
- Identified potential risks in unsecured transmissions
- Evaluated data exposure and network behavior
  
**Key Learnings**
- Network packet structure and communication flow
- PCAP file analysis using Wireshark
- Linux command-line operations for security tasks
- Basics of secure system design principles
- Importance of encryption and secure data handling
- Practical exposure to cybersecurity monitoring techniques
